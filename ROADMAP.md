# touchgrass — roadmap

Ordered feature list. Build top to bottom. Each item is a feature, not a phase — phases get grouped later in `checklist/`.

Architecture, stack rationale, and problem framing live in `HANDOFF.md` (decisions log) and will move into the README as the project matures. This file is just *what gets built and in what order*.

## Stack reference

| Component | Tech |
|---|---|
| Repo layout | Monorepo: `daemon/` (Python) + `app/` (React Native) |
| Daemon | Python 3.11+, Claude Code SDK (programmatic, headless) |
| Storage | SQLite |
| Transport | WebSocket (streaming) + REST (control) over Tailscale |
| Push | ntfy.sh |
| Phone | React Native, Android-first |
| Auth | `CLAUDE_CODE_OAUTH_TOKEN` (Claude Max subscription) + local PIN on phone |
| Sleep prevention | `caffeinate -i` (macOS) / `systemd-inhibit --what=sleep` (Linux) |

## Build order

### 1. Monorepo skeleton
Set up `daemon/` and `app/` as siblings under one repo. Root holds shared docs (`ROADMAP.md`, `HANDOFF.md`, `README.md`, `LICENSE`, `checklist/`). Each subproject has its own toolchain (`pyproject.toml` for daemon, RN's package.json for app) — no cross-build coupling.
**Acceptance:** `pip install -e ./daemon` and `cd app && npm install` both succeed on a clean clone.

### 2. Daemon config loader
Parse `~/.touchgrass/config.yaml`. Schema: list of projects (`name`, `path`), ntfy topic, optional pre-approved/pre-denied tool lists per project, optional permission timeout. Multi-project is first-class from day one — no single-project shortcut.
**Tech:** PyYAML, Pydantic for schema validation.
**Acceptance:** loader rejects malformed configs with clear errors; resolves `~` and relative paths; round-trips a multi-project example.

### 3. SQLite session store
Schema for sessions (id, project, created_at, status, goal), messages (session_id, role, content, timestamp, tool_calls), and pending permission requests. Migrations baked in from the start (even a hand-rolled version table is fine — just don't ship without one).
**Tech:** `sqlite3` stdlib or SQLAlchemy Core, depending on how the API shakes out.
**Acceptance:** can create/list/load sessions across daemon restarts; messages persist in order.

### 4. Claude Code SDK integration
Wire the SDK into a per-session runner. Each session is a Python coroutine that owns one SDK conversation, scoped to one project's working directory. Stream prompts in, stream responses (text + tool calls) out. Headless — no terminal.
**Tech:** Claude Code Python SDK, `asyncio`.
**Acceptance:** can fire a prompt at a session and see streamed assistant output land in the SQLite message log in real time.

### 5. WebSocket + REST API
- **REST:** list projects, list sessions, create session, get session detail, get message history.
- **WebSocket:** subscribe to a session — receive streamed assistant tokens, tool-call events, permission requests; send user prompts.
- Auth: bearer token in config (Tailscale already gates network reach; this is belt-and-suspenders).
**Tech:** FastAPI + `websockets`, or Starlette directly.
**Acceptance:** a CLI test client can list projects, start a session, stream a prompt, and watch tokens arrive.

### 6. Permission broker
Hook the SDK's `canUseTool` callback. Decision tree:
1. Tool in project's pre-approved list → allow.
2. Tool in project's pre-denied list → deny.
3. Otherwise → enqueue a permission request, push it over the WebSocket, block the session until a decision arrives or timeout fires.
4. Timeout / no client connected → deny (configurable default).
Decisions: **Allow once / Allow for project / Deny.** "Allow for project" mutates the in-memory pre-approved list for the rest of that project's daemon lifetime (persisted to config in a later item if we want it durable).
**Acceptance:** with a stub client that auto-approves, sessions run tools end-to-end; with no client, sessions deny after timeout.

### 7. ntfy.sh push
On each pending permission request and on session completion, POST to the configured ntfy topic. Include session id and a short action label so the phone can deep-link.
**Tech:** `httpx`.
**Acceptance:** subscribing to the topic from a browser shows pushes when permissions fire.

### 8. Per-session changelog writer
On session end (or on demand), write `<project_path>/.touchgrass/sessions/<session-id>.md` with: goal, status, summary, files touched, open threads, next steps. Generated by asking the session's own Claude to summarize before close. `.touchgrass/` added to a project-local gitignore on first write.
**Acceptance:** ending a session produces a markdown file readable enough to paste into a fresh Claude Code session and resume context.

### 9. Sleep-prevention docs + dev runner
Document `caffeinate -i` (macOS) and `systemd-inhibit --what=sleep` (Linux) prominently. Provide a `make dev` (or equivalent) target that wraps the daemon in the right invocation per OS.
**Acceptance:** `make dev` keeps the laptop awake on macOS; README has a clear "don't skip this" callout.

### 10. Tailscale setup guide
README section: install Tailscale on laptop and phone, log into the same tailnet, find the laptop's tailnet IP with `tailscale ip -4`, point the phone app at it.
**Acceptance:** a fresh user following the guide can reach the daemon from their phone without touching firewall settings.

### 11. RN app shell
Project skeleton, navigation, theming, env config (daemon URL, bearer token, ntfy topic stored locally).
**Tech:** React Native (bare or Expo — TBD when we get here), React Navigation.
**Acceptance:** app boots on Android, renders an empty home screen, reads/writes its config to local storage.

### 12. RN: PIN gate
Local PIN screen on app launch and after backgrounding past a threshold. Stored hashed in device keychain.
**Tech:** `react-native-keychain` or equivalent.
**Acceptance:** wrong PIN blocks app; correct PIN unlocks; no plaintext PIN at rest.

### 13. RN: project picker
Hits daemon's `GET /projects`, lists them, persists last-picked.
**Acceptance:** picking a project lands you on its session list.

### 14. RN: session list
Per-project list of sessions with status (active / waiting on permission / completed). Tap to open. "New session" button.
**Acceptance:** creating a session from the phone shows up in the list immediately and persists across app restarts.

### 15. RN: chat UI with streaming
Open a session → WebSocket subscribe → render streamed assistant tokens and tool-call events. Input box sends prompts. Scrollback hits the message-history REST endpoint.
**Acceptance:** prompts sent from the phone reach the daemon, responses stream back token-by-token, tool calls render inline.

### 16. RN: permission approval modal
When a permission request arrives over the WebSocket (or via ntfy deep-link), surface a modal with tool name, arguments, and three buttons: Allow once / Allow for project / Deny. Decision posts back to the daemon over the WebSocket.
**Acceptance:** approving from the phone unblocks the daemon session; denying causes the SDK to skip the tool.

### 17. RN: ntfy subscription
Subscribe to the configured ntfy topic for background push. Tapping a notification deep-links into the relevant session (and modal if it's a permission request).
**Tech:** `ntfy`'s native Android subscription, or `react-native-push-notification` against ntfy's webhook.
**Acceptance:** with the app backgrounded, a permission push wakes the phone and tapping it opens the right modal.

### 18. RN: file tree with AI summaries
Lazy-loaded directory tree for the active project (daemon exposes `GET /projects/:id/tree?path=...`). Tap a file → daemon asks the session's Claude for a one-paragraph summary, returned over REST and cached. Secondary action reveals raw contents.
**Acceptance:** tapping a file shows a summary within a few seconds; second tap on same file is instant (cache hit).

### 19. README pass
Write the user-facing README from scratch: what it is, the problem, install, configure, run, Tailscale guide, permission UX, desktop-resumption flow, FAQ, license. Lift the good parts of `OUTLINE_OLD.md` but drop the alpha/v0.1 framing.
**Acceptance:** a stranger can clone, follow the README, and end up with a working daemon-plus-phone setup.

## Later

Speculative or nice-to-have. Not on the build path until the items above are done and the tool is in real personal use.

- **iOS support** — port the RN app, requires Mac + iPhone for testing. PRs welcome.
- **Time-windowed permissions** — "Allow for next 30 minutes" as a fourth permission option.
- **Pattern-based auto-approval** — e.g. allow all `npm test`, deny all `rm -rf`. Per-project rules.
- **Persistent "Allow for project"** — write project-scoped allow decisions back to `config.yaml` so they survive daemon restarts.
- **VSCode extension** — natively reads `.touchgrass/sessions/` so desktop continuation feels first-class.
- **Repo auto-discovery** — scan common project directories and offer them in the picker instead of requiring explicit config.
- **Transcript export** — full session as shareable markdown.
- **Voice input** — hands-free prompting from the phone.
- **Polling fallback** — for users who don't want ntfy.
- **PWA web client** — for users who don't want to build the RN app.
